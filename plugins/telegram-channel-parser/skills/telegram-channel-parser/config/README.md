# Настройка telegram-channel-parser

## Быстрый старт

Скилл работает **без настройки** — парсит публичные каналы через веб-превью t.me/s/.

## Опциональная конфигурация

Создайте `config/.env` для удобства:

```bash
cp config/.env.example config/.env
```

### Параметры

| Переменная | Описание | По умолчанию |
|-----------|----------|--------------|
| `TG_CHANNELS` | Каналы для дайджеста (через запятую) | — |
| `TG_CACHE_TTL_HOURS` | Время жизни кеша в часах | `6` |

### Пример .env

```
TG_CHANNELS=countwithsasha,evilfreelancer,aostrikov_ai_agents,tips_ai,gonzo_ML
TG_CACHE_TTL_HOURS=6
```

## Дайджест-проект

Для ежедневных дайджестов:

1. Заполните `TG_CHANNELS` в `.env`
2. Запускайте: `bash scripts/digest.sh --period today`
3. Или настройте в Cowork: `/schedule` → "Каждый день в 21:00 собери дайджест каналов"

## Ограничения

- Только публичные каналы (с включённым t.me/s/ превью)
- Rate limit: 1.5с между запросами
- Пагинация: ~20 постов на страницу
- Некоторые каналы не показывают количество просмотров/пересылок
